# Mode INIT — scaffold a new base's structure

_On-demand procedure for kb's INIT mode. The shared model — the five-zone structure, routing test, memory cascade, project bindings, and Step 1 (Load context) — lives in [`SKILL.md`](../SKILL.md) and is already loaded; this file is the procedure only._

1. Create the zone skeleton — Calendar / Pillars / Resources / Streams, plus the inbound `+` and outbound `-` zones — and the root `Admin/MEMORY.md`.
2. Create the two `Admin/` subdivisions with stub index notes and governance documents:
   - `Admin/Governance/Governance.md` — title, purpose, link to decisions folder.
   - `Admin/Governance/Charter.md` — stub with headings: Purpose, Scope, Owner, Established.
   - `Admin/Governance/Conformance.md` — stub table: Skill | Adoption date | Notes.
   - `Admin/Operations/Operations.md` — title, purpose, link to activities index.
3. Copy the zone-scoped templates from `references/templates/` into a `Templates/` staging folder in the base (or a configured location), so the base owner has starter stubs for common note types. Do not create this folder if the base already has templates in place.
4. Seed each zone's note conventions (frontmatter, naming) from the structure model.
5. Hand ongoing capture to the SAVE / EXTRACT / DIGEST modes.
