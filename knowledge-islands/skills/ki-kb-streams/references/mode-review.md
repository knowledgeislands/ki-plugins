# Mode REVIEW — post-change review

_On-demand procedure for streams' REVIEW mode. The shared model — the zone-at-a-glance, the status lifecycle, the proposal anatomy, the bindings, Step 1, and the working rules + Enactment gate — lives in [`SKILL.md`](../SKILL.md) and is already loaded; this file is the procedure only._

After rollout, prepare an initial review summary (what went well, issues, lessons) and run it **interactively** — the summary is input, not output. Record the final review under a `## Post-Change Review` section (or in the process note if the lesson is structural), then set `status: reviewed`. Lessons may spawn new proposals.
